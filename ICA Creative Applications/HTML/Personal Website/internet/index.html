Internet Questions

How does the Internet work?

Invented by Al Gore, the Internet runs on magic. Well, alright, that may just be an Internet joke. The Internet in reality is pretty sophisticated. Let's start when you make a request to this current webpage: https://personal-website--darrenskidmore.repl.co/internet/, following the journey of one packet.

First, your client (Google Chrome) resolves the URL, by contacting the DNS record for repl.co. For this website, it will need to obtain a CNAME record first. Afterwards, the packet is sent. To paraphrase, it will travel in these steps:

  1. Your computer (localhost)
  2. The modem (192.168.0.1), whether that be a router, WAP, or cell tower
  3. The ISP's servers
  4. The third-party destination

If you requested an HTTPS url, it will also need to obtain a valid security certificate. Next, the server's backend code will generate the HTML response, and send it back to the client. The client may already have information cached, and this can only be bypassed by refreshing the page with CTRL+SHIFT+R.

What does a browser do with HTML/CSS/JS code?

HTML consists of the content of the webpage, CSS consists of styling instructions, and JS consists of Turing-complete code to modify HTML or CSS code (CSS is also technically Turing-complete, somehow). Interpreters in the browser process this codes. Some browsers, like the behemoth that is Internet Explorer, have outdated interpreters and do not recognize the most simple CSS attributes.

Cookies/Extensions

Cookies on the Internet are used to store local browser preferences or session IDs. They differ from LocalStorage and SessionStorage in that they are sent to the server via web requests, while the latter two are entirely client-side.

Server-side vs. Client-side

Server-side is used to describe anything that takes place on the webserver, while client-side is used to describe what happens to the local user.
